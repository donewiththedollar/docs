# Risk Management Best Practices

Successful algorithmic trading requires more than just good strategies‚Äîit demands **disciplined risk management**. This guide covers essential best practices for protecting your capital while maximizing long-term profitability.

---

## üéØ Core Principles

### 1. Never Risk More Than You Can Afford to Lose

<Warning>Only trade with capital you can afford to lose completely. Never use rent money, emergency funds, or borrowed capital for trading.</Warning>

**Recommended Allocation:**
- **Beginner:** 1-5% of investment portfolio
- **Intermediate:** 5-15% of investment portfolio
- **Advanced:** 15-30% of investment portfolio

### 2. Position Sizing is Everything

**The 1% Rule:**
No single symbol should represent more than 1-2% risk to your total trading capital.

```
Total Capital: $10,000
Max Risk per Symbol: $100-200 (1-2%)

If using 20x leverage:
Position size: $100 * 20 = $2,000 notional
Wallet exposure per symbol: 20% max
```

### 3. Diversification Reduces Risk

**Don't Put All Eggs in One Basket:**
- Trade 5-10 different symbols
- Mix volatility levels (BTC + altcoins)
- Use different strategies (Vortex DCA + XGrid)
- Spread across exchanges (if possible)

### 4. Leverage Amplifies Both Gains AND Losses

**Leverage Guidelines:**

| Experience | Max Leverage | Reason |
|------------|--------------|--------|
| Beginner | 5x-10x | Learn with safety margin |
| Intermediate | 10x-20x | Balance of profit and risk |
| Advanced | 20x-50x | High efficiency, requires skill |
| Expert | 50x+ | Extreme risk, tiny margin for error |

<Tip>Start with lower leverage (5x-10x) even if you're experienced. You can always increase it later.</Tip>

## üìä Position Management

### Total Exposure Limits

**Golden Rule:** Never exceed 80% total wallet exposure across all positions.

```json
{
  "symbols": ["BTCUSDT", "ETHUSDT", "SOLUSDT", "DOGEUSDT"],
  "wallet_exposure": 0.15  // 15% per symbol = 60% total
}
```

**Example Calculation:**
```
Balance: $10,000
Symbols: 4
Exposure per symbol: 15%

Max position value per symbol: $10,000 * 0.15 = $1,500
Total exposure: $1,500 * 4 = $6,000 (60%)
Reserve: $4,000 (40%) ‚úÖ Safe
```

### Reserve Capital

**Always maintain reserves:**
- **20% minimum:** For averaging down
- **30% recommended:** For opportunities and safety
- **40% conservative:** Maximum safety buffer

**Why Reserves Matter:**
```
Scenario WITHOUT reserve:
100% deployed ‚Üí Price drops 10%
‚Üí No capital to average down
‚Üí Forced to wait or take loss

Scenario WITH 40% reserve:
60% deployed ‚Üí Price drops 10%
‚Üí Use 20% reserve to average down
‚Üí Improve entry price, faster recovery
```

## üìâ Drawdown Management

### Understanding Drawdown

**Drawdown** = Peak-to-trough decline in account value

```
Account Peak:   $10,000
Current Value:  $9,200
Drawdown:       $800 (8%)
```

### Acceptable Drawdown Limits

| Risk Tolerance | Max Drawdown | Action Plan |
|----------------|--------------|-------------|
| Conservative | 10-15% | Stop trading, reassess |
| Moderate | 15-25% | Reduce exposure 50% |
| Aggressive | 25-35% | Reduce exposure 25% |
| Very Aggressive | 35-50% | Continue with caution |

### Recovery from Drawdowns

**The Math of Recovery:**
```
10% loss requires 11.1% gain to recover
20% loss requires 25% gain to recover
30% loss requires 42.8% gain to recover
50% loss requires 100% gain to recover
```

<Warning>Deep drawdowns become exponentially harder to recover from. Cut losses early!</Warning>

### Virtual Chunking for Recovery

Enable virtual chunking to systematically recover from underwater positions:

```json
{
  "virtual_chunking_enabled": true,
  "chunking_threshold": 0.03,  // Activate at -3% unrealized loss
  "chunk_size": 0.25,  // Split into 4 chunks
  "attack_order_multiplier": 3.0  // 3x size for averaging down
}
```

**How It Works:**
```
Position: -$300 (3% loss) ‚Üí Virtual chunking activates
Split into 4 chunks:
  Chunk 1: -$75 ‚Üí Place 3x attack order
  TP at 0.1% ‚Üí Profit $7.50, chunk closed
  Chunk 2: -$75 ‚Üí Repeat
  ...
Total position gradually recovered
```

### Daily Checks

- [ ] Review total account balance

- [ ] Scan logs for errors

- [ ] Monitor exchange balance

### Weekly Analysis

**Every Week:**
- [ ] Calculate weekly profit/loss %
- [ ] Review grid performance per symbol
- [ ] Analyze hedge cycles (if using auto-hedge)
- [ ] Adjust configurations based on results
- [ ] Check exchange fees paid

### Monthly Review

**Every Month:**
- [ ] Total return vs. target
- [ ] Sharpe ratio / risk-adjusted return
- [ ] Max drawdown experienced
- [ ] Strategy performance comparison
- [ ] Rebalance portfolio if needed

### Key Metrics to Track

```
ROI (Return on Investment):
  = (Current Value - Initial Value) / Initial Value * 100

Sharpe Ratio (Risk-Adjusted Return):
  = (Average Return - Risk-Free Rate) / Standard Deviation

Max Drawdown:
  = (Peak Value - Trough Value) / Peak Value * 100

Win Rate:
  = Winning Trades / Total Trades * 100

Profit Factor:
  = Gross Profit / Gross Loss
```

## üåü Advanced Risk Techniques

### Kelly Criterion (Position Sizing)

**Formula:**
```
Kelly % = (Win Rate * Avg Win - Loss Rate * Avg Loss) / Avg Win

Example:
Win Rate: 60%
Avg Win: 1%
Loss Rate: 40%
Avg Loss: 0.8%

Kelly % = (0.6 * 1 - 0.4 * 0.8) / 1 = 0.28 (28%)
```

Use 25-50% of Kelly result for safety:
```
Kelly: 28%
Conservative Kelly: 28% * 0.25 = 7% per position
```

### Correlation-Based Diversification

**Avoid Correlated Positions:**
```
‚ùå Bad Diversification:
  DOGEUSDT
  SHIBUSDT
  PEPEUSDT
‚Üí All meme coins, highly correlated

‚úÖ Good Diversification:
  BTCUSDT (store of value)
  ETHUSDT (smart contract platform)
  SOLUSDT (high-performance L1)
  DOGEUSDT (meme coin)
‚Üí Different sectors, lower correlation
```

### Dynamic Position Sizing

Adjust exposure based on performance:

```python
if monthly_return > 10%:
    # Winning ‚Üí Increase exposure slightly
    wallet_exposure *= 1.1  # +10%

elif monthly_return < -10%:
```
